
# Tales From The Spiral
 
**Live Site:** http://classwork.engr.oregonstate.edu:7890
 
Our project is a game and wiki-style database for *Tales From The Spiral*. The database is designed to organize game world information using core entity tables and intersection tables. The main entity tables are Areas, NPCs, Enemies, Abilities, and Hazards. The intersection tables model the many-to-many relationships between these entities. The goal of the project is to make the world data easier to browse, manage, and update through a web UI and supporting SQL queries. This final version continues the same project direction from the earlier steps while keeping the schema and UI plan aligned.
 
---
 
## Database Outline
 
The database uses the following entities and attributes, with intersection tables for M:N relationships:
 
### Areas
Stores locations within the game's setting that can be traveled to by a player, which can contain characters both friendly and hostile.
 
| Attribute | Type | Constraints |
|---|---|---|
| areaID | varchar(255) | PK, unique, not NULL |
| areaLevel | int | not NULL |
 
**Relationships:**
- One Area can have many NPCs
- One NPC must belong to one Area
- Many Areas can contain many Enemies
- Many Areas can contain many Hazards
---
 
### NPCs
Stores non-player characters that the player can interact with, such as merchants, quest givers, or townspeople.
 
| Attribute | Type | Constraints |
|---|---|---|
| npcID | varchar(255) | PK, unique, not NULL |
| hp | int | not NULL |
| shopkeeper | bool | not NULL |
| quest | bool | not NULL |
| enemyID | varchar(255) | FK |
 
**Relationships:**
- Many NPCs belong to one Area
- Many NPCs can have many Abilities
- Each NPC may have one Enemy representing them
---
 
### Enemies
Stores hostile characters or monsters the player may fight.
 
| Attribute | Type | Constraints |
|---|---|---|
| enemyID | varchar(255) | PK, unique, not NULL |
| hp | int | not NULL |
| damage | int | not NULL |
 
**Relationships:**
- Many Enemies can appear in many Areas
- Many Enemies can have many Abilities
---
 
### Abilities
Stores actions, powers, or traits used by NPCs or Enemies, such as attacks, spells, or support actions.
 
| Attribute | Type | Constraints |
|---|---|---|
| abilityID | varchar(255) | PK, unique, not NULL |
| damage | int | nullable |
 
**Relationships:**
- Many Abilities can belong to many NPCs
- Many Abilities can belong to many Enemies
---
 
### Hazards
Stores environmental dangers found in areas, such as traps or ambushes.
 
| Attribute | Type | Constraints |
|---|---|---|
| hazardID | varchar(255) | PK, unique, not NULL |
| damage | int | not NULL |
 
**Relationships:**
- Many Hazards can appear in many Areas
---
 
## Relationship Summary
 
| Relationship | Type |
|---|---|
| Areas → NPCs | 1:M |
| Areas → Enemies | M:N |
| Areas → Hazards | M:N |
| NPCs → Abilities | M:N |
| Enemies → Abilities | M:N |
